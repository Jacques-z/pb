# 设计：Android 客户端

## 背景
需要在移动端提供与桌面一致的排班客户端能力，并确保后台提醒可用。用户要求复用现有前端并使用 Tauri 打包。

## 目标 / 非目标
- 目标
  - Android 客户端可登录并访问既有模块（即将班次、管理控制台等）
  - 后台常驻并触发本地提醒
  - 沿用现有 HTTP JSON API 与鉴权
  - 通过 GitHub Actions 打包调试版 APK 并发布 Release
- 非目标
  - iOS 版本
  - 云端推送与远程通知
  - 全新原生 UI 重写

## 决策
- 采用 Tauri 移动端能力打包现有前端 UI
- Android 端继续使用本地通知，保持与桌面一致的提醒逻辑
- 不引入新的服务端接口，全部复用当前 API
- GitHub Actions 在 tag 触发时打包调试版 APK 并发布到 Release

## 备选方案
- 原生 Android 客户端
- 跨平台框架重写（Flutter/React Native）
- 手动本地打包或自托管 CI

## 风险与权衡
- Android 后台限制导致通知可能被系统压制，需要提示用户允许后台运行
- 移动端屏幕尺寸与交互方式可能影响管理控制台体验
- CI 环境依赖较重，可能增加构建时间或失败率

## 迁移计划
无服务端迁移；仅新增客户端产物与配置。

## 开放问题
无。
